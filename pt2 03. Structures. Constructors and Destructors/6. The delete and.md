Разберём ключевые слова **`= delete`** и **`= default`** для конструкторов — это управление автоматически генерируемыми функциями.

---

# 1️⃣ Что означает `= default`?

```cpp
class A {
public:
    A() = default;
};
```

`= default` означает:

> Попросить компилятор сгенерировать стандартную реализацию этой функции.

То есть:

- не писать свою реализацию
    
- использовать поведение по умолчанию
    
- но при этом явно объявить её
    

---

## Когда это нужно?

### 🔹 1. Если объявлен другой конструктор

Если есть пользовательский конструктор:

```cpp
class A {
public:
    A(int x) {}
};
```

Компилятор **не создаёт** default constructor.

Чтобы вернуть его:

```cpp
class A {
public:
    A(int x) {}
    A() = default;
};
```

---

### 🔹 2. Для явного указания намерения

Иногда пишут:

```cpp
A(const A&) = default;
A(A&&) = default;
```

Это показывает:

- мы хотим стандартное копирование/перемещение
    
- и ничего особенного не делаем
    

---

# 2️⃣ Что означает `= delete`?

```cpp
class A {
public:
    A() = delete;
};
```

`= delete` означает:

> Запретить использование этой функции.

Компилятор не позволит её вызвать.

---

# 3️⃣ Когда используют `= delete`?

---

## 🔹 1. Запрет создания объекта без параметров

```cpp
class A {
public:
    A() = delete;
    A(int x) {}
};
```

Теперь:

```cpp
A a;      // ❌ ошибка
A b(10);  // ✅
```

---

## 🔹 2. Запрет копирования

```cpp
class A {
public:
    A(const A&) = delete;
};
```

Теперь:

```cpp
A a;
A b = a;  // ❌ ошибка
```

Используется в:

- singleton
    
- классах с уникальным ресурсом
    

---

## 🔹 3. Запрет перемещения

```cpp
A(A&&) = delete;
```

---

## 🔹 4. Запрет неявных преобразований

```cpp
class A {
public:
    A(int) = delete;
};
```

---

# 4️⃣ Разница между `= default` и не писать ничего

### Если не писать ничего:

- компилятор может создать функцию
    
- а может удалить её автоматически
    

### Если написать `= default`:

- вы явно просите компилятор создать её
    
- если он не может — будет ошибка компиляции
    

Это делает поведение более контролируемым.

---

# 5️⃣ Пример сравнения

```cpp
class A {
    const int x;
};
```

Компилятор попытается создать default constructor → он будет deleted.

Если написать:

```cpp
A() = default;
```

Компилятор явно покажет ошибку, потому что `const x` нельзя инициализировать.

---

# 6️⃣ Важный момент: где можно писать `= default`?

Можно:

- в объявлении внутри класса
    
- вне класса
    

```cpp
class A {
public:
    A();
};

A::A() = default;
```

---

# 7️⃣ Типичные применения

|Ситуация|Что использовать|
|---|---|
|Вернуть default constructor|`= default`|
|Запретить создание объекта|`= delete`|
|Запретить копирование|`= delete`|
|Запретить перемещение|`= delete`|
|Явно указать стандартное поведение|`= default`|

---

# 8️⃣ Краткое резюме

## `= default`

- говорит компилятору создать стандартную реализацию
    
- делает намерение явным
    
- может привести к ошибке, если генерация невозможна
    

## `= delete`

- запрещает использование функции
    
- используется для ограничения поведения класса
    
- особенно часто применяется для копирования/перемещения
