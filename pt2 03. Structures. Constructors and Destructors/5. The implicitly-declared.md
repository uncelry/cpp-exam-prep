Разберём тему **неявно объявленного (implicitly-declared) конструктора по умолчанию** — это важный раздел правил генерации специальных функций в C++.

---

# 1️⃣ Что такое implicitly-declared (defaulted) constructor?

Это **конструктор по умолчанию**, который:

- компилятор объявляет автоматически
    
- если пользователь не объявил ни одного конструктора
    
- и если нет причин его удалить
    

Фактически компилятор создаёт:

```cpp
A() = default;
```

---

# 2️⃣ Когда он генерируется автоматически?

Компилятор объявляет default constructor, если:

✔ В классе **нет ни одного пользовательского конструктора**

Пример:

```cpp
class A {
    int x;
};
```

Компилятор создаёт:

```cpp
A() = default;
```

---

# 3️⃣ Что делает автоматически сгенерированный конструктор?

Он:

- вызывает конструкторы базовых классов
    
- вызывает конструкторы всех полей-членов
    
- для встроенных типов ничего не делает (они остаются неинициализированными)
    

Пример:

```cpp
class A {
    int x;
};
```

```cpp
A a;  // x неинициализирован!
```

---

# 4️⃣ Когда он НЕ генерируется?

## 🔹 1. Если объявлен любой пользовательский конструктор

```cpp
class A {
public:
    A(int x) {}
};
```

Теперь:

```cpp
A a;  // ❌ ошибки — default constructor не существует
```

Компилятор больше не создаёт его автоматически.

---

## 🔹 2. Если его нельзя сгенерировать корректно

Даже если он объявлен, он может быть **удалён (deleted)**.

Это происходит если:

### ❌ Есть поле без конструктора по умолчанию

```cpp
class B {
public:
    B(int x) {}
};

class A {
    B b;   // у B нет default constructor
};
```

Компилятор пытается создать `A()`, но:

- он должен вызвать `B()`
    
- а `B()` не существует
    

→ `A()` будет implicitly-declared as deleted

---

### ❌ Есть ссылка без инициализации

```cpp
class A {
    int& ref;
};
```

Ссылка должна быть инициализирована.

→ default constructor будет удалён

---

### ❌ Есть const поле без инициализации

```cpp
class A {
    const int x;
};
```

Const поле нельзя оставить неинициализированным.

→ default constructor удаляется

---

# 5️⃣ Разница между "не объявлен" и "удалён"

- Не объявлен — если есть пользовательский конструктор.
    
- Удалён — если компилятор не может корректно его сгенерировать.
    

---

# 6️⃣ Можно ли принудительно его вернуть?

Да:

```cpp
class A {
public:
    A(int x) {}
    A() = default;
};
```

Но если есть проблемы (const, reference, члены без default constructor), он всё равно будет удалён.

---

# 7️⃣ Что значит `= default`?

Это означает:

> использовать стандартную реализацию компилятора.

Можно писать:

```cpp
A() = default;
```

чтобы явно попросить компилятор создать его.

---

# 8️⃣ Когда default constructor считается trivial?

Он будет **trivial**, если:

- нет пользовательских конструкторов
    
- нет виртуальных функций
    
- все поля имеют trivial constructors
    
- нет виртуального базового класса
    

Trivial constructor — ничего не делает на уровне кода.

---

# 9️⃣ Пример анализа

```cpp
class A {
    int x;
};
```

✔ default constructor генерируется  
✔ он trivial

---

```cpp
class A {
    std::string s;
};
```

✔ default constructor генерируется  
❌ он не trivial (вызывает `std::string()`)

---

```cpp
class A {
    const int x;
};
```

❌ default constructor будет deleted

---

# 🔟 Итоговая таблица

|Ситуация|Что происходит|
|---|---|
|Нет конструкторов|Генерируется автоматически|
|Есть любой пользовательский конструктор|Не генерируется|
|Есть const без инициализации|Deleted|
|Есть reference без инициализации|Deleted|
|Член без default constructor|Deleted|

---

# 1️⃣1️⃣ Краткое резюме

- Компилятор автоматически объявляет default constructor, если нет пользовательских
    
- Он может быть автоматически удалён
    
- Нельзя создать default constructor, если члены не могут быть корректно инициализированы
    
- Можно явно написать `= default`
    
- Trivial constructor ничего не делает
