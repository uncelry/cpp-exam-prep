
# 1️⃣ Что такое copy constructor?

**Конструктор копирования** — это специальный конструктор, который создаёт новый объект как копию существующего.

### Правильная сигнатура:

```cpp
ClassName(const ClassName& other);
```

Пример:

```cpp
class A {
public:
    A(const A& other) {
        std::cout << "Copy constructor\n";
    }
};
```

---

# 2️⃣ Когда он вызывается?

Copy constructor вызывается:

✔ При инициализации объектом того же типа:

```cpp
A a1;
A a2 = a1;
A a3(a1);
```

✔ При передаче по значению:

```cpp
void func(A obj);
```

✔ При возврате по значению:

```cpp
A make();
```

---

# 3️⃣ Генерируется ли он всегда автоматически?

❗ Нет, не всегда.

---

## 🔹 Когда компилятор генерирует copy constructor?

Если:

- вы его не объявили
    
- и нет причин его удалить
    

Тогда компилятор создаёт:

```cpp
A(const A&) = default;
```

---

# 4️⃣ Что делает автоматически сгенерированный copy constructor?

Он выполняет **memberwise copy**:

- копирует базовые классы
    
- копирует все поля
    
- делает поверхностную (shallow) копию
    

Пример:

```cpp
class A {
    int x;
    std::string s;
};
```

Компилятор создаст копирование:

- `x` копируется
    
- `s` вызывает свой copy constructor
    

---

# 5️⃣ Когда copy constructor НЕ генерируется или удаляется?

## 🔴 1. Если объявлен move constructor

Если вы объявили move constructor или move assignment operator:

```cpp
class A {
public:
    A(A&&) {}
};
```

→ copy constructor будет **удалён**

---

## 🔴 2. Если один из членов не копируем

```cpp
class B {
public:
    B(const B&) = delete;
};

class A {
    B b;
};
```

→ copy constructor у `A` будет deleted.

---

## 🔴 3. Если вы сами его удалили

```cpp
A(const A&) = delete;
```

---

# 6️⃣ Как правильно объявлять copy constructor?

Правильная форма:

```cpp
class A {
public:
    A(const A& other);
};
```

Важно:

- аргумент — `const A&`
    
- не по значению (иначе бесконечная рекурсия!)
    
- не без const (ограничит копирование временных объектов)
    

---

# 7️⃣ Когда обязательно нужно писать свой copy constructor?

Когда класс управляет ресурсом:

- сырые указатели
    
- динамическая память
    
- файловые дескрипторы
    
- сокеты
    
- мьютексы
    

---

## Пример неправильного поведения (shallow copy)

```cpp
class A {
    int* data;
public:
    A(int value) {
        data = new int(value);
    }

    ~A() {
        delete data;
    }
};
```

Если не определить copy constructor:

```cpp
A a1(10);
A a2 = a1;  // оба объекта указывают на один и тот же int
```

При уничтожении будет double delete → UB.

---

## Правильная реализация (deep copy)

```cpp
class A {
    int* data;
public:
    A(int value) {
        data = new int(value);
    }

    A(const A& other) {
        data = new int(*other.data);
    }

    ~A() {
        delete data;
    }
};
```

Теперь каждый объект владеет своей памятью.

---

# 8️⃣ Правило трёх

Если класс:

- определяет destructor
    
- управляет ресурсом
    

То почти всегда нужно определить:

- copy constructor
    
- copy assignment operator
    
- destructor
    

(в современном C++ — правило пяти)

---

# 9️⃣ Что если не определить copy constructor?

Компилятор:

- либо создаст поверхностную копию
    
- либо удалит его
    
- либо сделает его `= default`
    

---

# 🔟 Краткая таблица

|Ситуация|Что происходит|
|---|---|
|Нет пользовательских специальных функций|Copy constructor генерируется|
|Есть move constructor|Copy constructor удаляется|
|Член не копируем|Copy constructor удаляется|
|Класс управляет ресурсом|Нужно писать вручную|

---

# 1️⃣1️⃣ Краткое резюме

- Copy constructor создаёт объект-копию
    
- Компилятор генерирует его автоматически
    
- Он выполняет memberwise copy
    
- Может быть автоматически удалён
    
- Нужно писать вручную при управлении ресурсами
    
- Правильная сигнатура: `ClassName(const ClassName&)`
