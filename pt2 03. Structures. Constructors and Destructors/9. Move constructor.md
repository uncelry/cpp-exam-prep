
# 1️⃣ Что такое move constructor?

**Move constructor** — это специальный конструктор, который:

- принимает **rvalue-ссылку** (`T&&`)
    
- "забирает" ресурсы у временного объекта
    
- вместо копирования выполняет перемещение
    

### Сигнатура:

```cpp
ClassName(ClassName&& other);
```

---

# 2️⃣ Для чего он нужен?

Главная цель — **эффективность**.

Вместо:

- копирования динамической памяти
    
- копирования буферов
    
- дублирования ресурсов
    

мы просто:

- передаём указатель
    
- обнуляем старый объект
    

---

## Пример без move

```cpp
std::vector<int> v1 = {1,2,3};
std::vector<int> v2 = v1;  // копирование (вся память копируется)
```

---

## Пример с move

```cpp
std::vector<int> v2 = std::move(v1);  // перемещение
```

- память не копируется
    
- указатель просто передаётся
    
- операция быстрая
    

---

# 3️⃣ Когда вызывается move constructor?

✔ При инициализации временным объектом:

```cpp
A a = A();
```

✔ При использовании `std::move`:

```cpp
A a1;
A a2 = std::move(a1);
```

✔ При возврате временного объекта:

```cpp
A create() {
    return A();
}
```

---

# 4️⃣ Как понять, что объект “moved-from”?

После перемещения:

- объект остаётся **валидным**
    
- но его состояние **неопределённое (unspecified)**
    
- он не должен использоваться, кроме:
    
    - уничтожения
        
    - присваивания нового значения
        

---

## Пример

```cpp
std::string s1 = "hello";
std::string s2 = std::move(s1);
```

После этого:

- `s2` содержит "hello"
    
- `s1` — валиден, но обычно пустой
    

⚠ Стандарт гарантирует только валидность, а не конкретное состояние.

---

# 5️⃣ Как правильно реализовать move constructor?

Основные шаги:

1. Переместить ресурсы
    
2. Обнулить источник
    

---

## Пример правильной реализации

```cpp
class A {
    int* data;
public:
    A(int value) {
        data = new int(value);
    }

    // Move constructor
    A(A&& other) noexcept {
        data = other.data;      // забираем ресурс
        other.data = nullptr;   // обнуляем источник
    }

    ~A() {
        delete data;
    }
};
```

---

# 6️⃣ Почему нужно обнулять источник?

Если не обнулить:

- оба объекта будут указывать на одну и ту же память
    
- произойдёт double delete
    

---

# 7️⃣ Почему рекомендуется noexcept?

```cpp
A(A&& other) noexcept;
```

Стандартные контейнеры (например, `std::vector`) используют move только если он `noexcept`.

Если move может бросить исключение — будет использовано копирование.

---

# 8️⃣ Что происходит, если move constructor не определён?

Если:

- он не объявлен
    
- но нет пользовательского copy constructor или destructor
    

→ компилятор может сгенерировать его автоматически.

Но если:

- объявлен copy constructor
    
- объявлен destructor
    

→ move constructor НЕ будет сгенерирован.

---

# 9️⃣ Правило пяти

Если класс управляет ресурсом и вы пишете:

- destructor
    
- copy constructor
    

То обычно нужно написать и:

- move constructor
    
- move assignment operator
    

---

# 🔟 Полный пример с ресурсом

```cpp
class Buffer {
    int* data;
    size_t size;

public:
    Buffer(size_t s) : size(s) {
        data = new int[size];
    }

    // Move constructor
    Buffer(Buffer&& other) noexcept
        : data(other.data), size(other.size)
    {
        other.data = nullptr;
        other.size = 0;
    }

    ~Buffer() {
        delete[] data;
    }
};
```

---

# 1️⃣1️⃣ Важные моменты

✔ Move работает только с rvalue  
✔ `std::move` не перемещает — он просто приводит к rvalue  
✔ Объект после move валиден, но состояние не гарантируется  
✔ Всегда обнулять источник  
✔ Желательно писать `noexcept`

---

# 1️⃣2️⃣ Краткое резюме

Move constructor:

- нужен для эффективной передачи ресурсов
    
- принимает `T&&`
    
- забирает ресурсы и обнуляет источник
    
- вызывается для rvalue
    
- после перемещения объект валиден, но в неопределённом состоянии
    
- важен для производительности контейнеров
