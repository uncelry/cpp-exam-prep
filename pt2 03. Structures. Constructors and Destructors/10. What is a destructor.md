Разберём тему **деструктора (destructor)** — одну из самых важных в управлении ресурсами.

---

# 1️⃣ Что такое деструктор?

**Деструктор** — это специальная функция-член класса, которая:

- вызывается автоматически при уничтожении объекта
    
- освобождает ресурсы
    
- имеет имя `~ClassName`
    
- не имеет параметров
    
- не имеет возвращаемого типа
    

### Пример:

```cpp
class A {
public:
    ~A() {
        std::cout << "Destructor called\n";
    }
};
```

---

# 2️⃣ Когда вызывается деструктор?

Деструктор вызывается автоматически:

### 🔹 1. Для локальных объектов

В конце области видимости:

```cpp
void func() {
    A a;
}  // здесь вызывается ~A()
```

---

### 🔹 2. Для динамических объектов

```cpp
A* p = new A();
delete p;  // вызывает ~A()
```

---

### 🔹 3. Для временных объектов

В конце полного выражения:

```cpp
A();
```

---

### 🔹 4. Для элементов контейнеров

Когда контейнер уничтожается или перераспределяется.

---

# 3️⃣ Что происходит при уничтожении объекта?

Порядок:

1️⃣ Выполняется тело деструктора  
2️⃣ Уничтожаются поля класса (в обратном порядке объявления)  
3️⃣ Вызывается деструктор базового класса

Важно:

> Порядок уничтожения — обратный порядку создания.

---

# 4️⃣ Что должен делать деструктор?

- освобождать динамическую память
    
- закрывать файлы
    
- освобождать сокеты
    
- освобождать мьютексы
    
- освобождать другие ресурсы
    

---

## Пример управления ресурсом:

```cpp
class A {
    int* data;
public:
    A() {
        data = new int(10);
    }

    ~A() {
        delete data;
    }
};
```

Без деструктора → утечка памяти.

---

# 5️⃣ Почему деструктор должен быть virtual в базовом классе?

Это **критически важно при полиморфизме**.

---

## Пример:

```cpp
class Base {
public:
    ~Base() {
        std::cout << "Base destructor\n";
    }
};

class Derived : public Base {
public:
    ~Derived() {
        std::cout << "Derived destructor\n";
    }
};
```

Использование:

```cpp
Base* ptr = new Derived();
delete ptr;
```

---

## Что произойдёт?

Если деструктор НЕ virtual:

- вызовется только `~Base()`
    
- `~Derived()` НЕ вызовется
    
- ресурсы Derived НЕ освободятся
    
- возникает утечка памяти
    
- это Undefined Behavior
    

---

# 6️⃣ Правильный вариант

```cpp
class Base {
public:
    virtual ~Base() {
        std::cout << "Base destructor\n";
    }
};
```

Теперь:

```cpp
Base* ptr = new Derived();
delete ptr;
```

Вызовется:

1. `~Derived()`
    
2. `~Base()`
    

Правильно и безопасно.

---

# 7️⃣ Почему так происходит?

Без `virtual`:

- выбор функции происходит на этапе компиляции
    
- используется тип указателя (`Base*`)
    

С `virtual`:

- используется динамическая диспетчеризация
    
- вызывается деструктор реального типа объекта
    

---

# 8️⃣ Золотое правило

> Если класс предназначен для полиморфного использования (имеет virtual-функции), его деструктор должен быть virtual.

---

# 9️⃣ Что если деструктор protected?

Иногда делают:

```cpp
class Base {
protected:
    ~Base() {}
};
```

Это запрещает удаление через указатель на базовый класс.

---

# 🔟 Краткое резюме

### Деструктор:

- вызывается автоматически при уничтожении объекта
    
- освобождает ресурсы
    
- выполняется перед уничтожением полей и базовых классов
    

### Порядок:

- сначала тело деструктора
    
- потом поля (в обратном порядке)
    
- потом базовый класс
    

### Virtual destructor:

- обязателен для полиморфных базовых классов
    
- иначе возникает UB
    
- без него Derived destructor не вызывается
    

---

# 1️⃣1️⃣ Главное правило

Если класс имеет хотя бы одну virtual-функцию → его деструктор должен быть virtual.
