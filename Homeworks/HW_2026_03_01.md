## Задача 1. Делегирующие конструкторы

Написать класс `Person`, который:

* хранит `std::string name` и `int age`
* имеет **три конструктора**:

  1. без параметров
  2. с одним параметром (только имя)
  3. с двумя параметрами (имя и возраст)

Требования:

* Конструкторы без параметров и с одним параметром должны делегировать главному конструктору (с двумя параметрами).
* Каждый конструктор должен выводить в консоль сообщение о своём вызове.
* Запретить создание объекта без параметров с помощью `=delete`.
* Явно объявить copy constructor как `=default`.
* В `main()` создать несколько объектов разными способами и продемонстрировать порядок вызовов.

## Задача 2. Converting constructor + explicit

Написать класс `Temperature`, который:

* хранит температуру в градусах Цельсия (`double celsius`)
* имеет конструктор с одним параметром `double`

Требования:

1. Сначала реализовать конструктор без `explicit`.
2. В `main()` продемонстрировать:

   * неявное преобразование из `double`
   * передачу числа в функцию, принимающую `Temperature`
3. Затем добавить `explicit`.
4. Исправить код так, чтобы он снова компилировался.
5. Добавить copy constructor, который выводит `"Copy constructor"`.

Показать в `main()`, когда вызывается copy constructor.

## Задача 3. Класс с ресурсом: copy constructor

Написать класс `StringBuffer`, который:

* хранит `char* data`
* выделяет динамическую память в конструкторе
* освобождает её в деструкторе

Требования:

1. Реализовать:

   * конструктор с параметром `const char*`
   * деструктор
2. Реализовать корректный copy constructor (deep copy).
3. В `main()`:

   * создать объект
   * скопировать его
   * продемонстрировать, что данные независимы.
4. Явно запретить move constructor с помощью `=delete`.

## Задача 4. Реализация move constructor

На основе предыдущего класса создать новый класс `DynamicArray`, который:

* хранит `int* data`
* хранит размер массива
* выделяет память в конструкторе

Требования:

1. Реализовать:

   * конструктор
   * деструктор
   * copy constructor (deep copy)
   * move constructor (с обнулением источника)
2. Move constructor должен быть `noexcept`.
3. В `main()`:

   * создать объект
   * переместить его через `std::move`
   * показать, что исходный объект остаётся валидным, но "пустым".
4. Явно объявить copy constructor через `=default` и объяснить в комментарии, почему в данном случае так делать нельзя (но оставить рабочую реализацию).

## Задача 5. Полиморфизм и виртуальный деструктор

Написать иерархию:

* Базовый класс `Shape`
* Производный класс `Circle`
* Производный класс `Rectangle`

Требования:

1. В базовом классе:

   * виртуальный метод `area()`
   * виртуальный деструктор (с выводом сообщения)
2. В производных классах:

   * динамически выделяемый ресурс (например, указатель на double)
   * собственный деструктор (с выводом)
3. В `main()`:

   * создать массив указателей `Shape*`
   * положить туда объекты `Circle` и `Rectangle`
   * удалить их через `delete`
4. В комментарии написать, что произойдёт, если убрать `virtual` у деструктора базового класса.

---

### Общие требования:

* Использовать `std::cout` для отслеживания порядка вызовов.
* Обязательно демонстрировать вызовы copy и move.
* Везде, где возможно, показывать работу `=delete` и `=default`.
* Код должен компилироваться.
* Все ресурсы должны освобождаться корректно (без утечек).
* В комментариях кратко пояснять, какой механизм демонстрируется.
