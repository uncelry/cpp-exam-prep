
The void type

**Тип `void` (The void type)**

---

## 1️⃣ Перевод темы

**The void type → Тип `void` (тип «пустота»)**

---

## 2️⃣ Что такое `void`

`void` — это специальный фундаментальный тип в C++, который означает **отсутствие значения или типа**.

Проще говоря:

> `void` говорит компилятору: «здесь **нет данных**».

---

## 3️⃣ Основные случаи использования `void`

### 1️⃣ `void` как тип возвращаемого значения функции

Функция с типом `void` **ничего не возвращает**.

Пример:

```cpp
void printHello() {
    std::cout << "Hello\n";
}
```

Такая функция:

- выполняет действия
- не возвращает результат через `return`

Можно писать:

```cpp
return;
```

но нельзя:

```cpp
return 5; // ошибка
```

---

### 2️⃣ `void` в списке параметров функции

Используется (редко) для обозначения, что функция **не принимает аргументов**.

```cpp
void foo(void);
```

В современном C++ чаще пишут:

```cpp
void foo();
```

---

### 3️⃣ Указатель `void*`

`void*` — это **универсальный указатель**, который может хранить адрес объекта любого типа.

Пример:

```cpp
int x = 10;
void* p = &x;
```

⚠️ Особенности:

- `void*` **нельзя разыменовывать напрямую**
- нужно привести к конкретному типу

```cpp
int* ip = static_cast<int*>(p);
std::cout << *ip;
```

---

### 4️⃣ Отличие `void*` от указателей других типов

```cpp
int* p1;
void* p2;
```

- `int*` знает размер объекта (`sizeof(int)`)
- `void*` **не знает**, на сколько байт указывает

Поэтому запрещено:

```cpp
p2++; // ошибка компиляции
```

---

## 4️⃣ `void` и выражения

❌ Нельзя создавать переменные типа `void`:

```cpp
void x; // ошибка
```

❌ Нельзя использовать `void` как обычный тип данных:

```cpp
void y = 5; // ошибка
```

---

## 6️⃣ `void` и возвращаемые выражения

Функции `void`:

- можно вызывать как инструкцию
- нельзя использовать в выражениях

```cpp
printHello();      // ок
int x = printHello(); // ошибка
```

---

## 7️⃣ Где `void` используется на практике

✔ функции без возвращаемого значения  
✔ универсальные интерфейсы (C API)  
✔ низкоуровневое программирование  
✔ работа с памятью  
✔ callback-функции

Но в современном C++:

- чаще используют шаблоны
- `std::variant`, `std::any`
- умные указатели

---

## 8️⃣ Итог

Надо понимать:

✔ `void` означает «нет значения»  
✔ функции `void` ничего не возвращают  
✔ `void*` — универсальный указатель  
✔ `void*` нельзя разыменовывать без приведения  
✔ переменных типа `void` не существует  
✔ в современном C++ `void*` используют ограниченно
