---

## 2️⃣ Что такое `bool` в C++

`bool` — это фундаментальный тип данных, предназначенный для хранения **логических значений**.

Возможные значения:

```cpp
true   // истина
false  // ложь
```

Размер:

- по стандарту: не меньше 1 байта
- на практике: **1 байт**

Пример:

```cpp
bool isReady = true;
bool isEmpty = false;
```

---

## 3️⃣ Связь `bool` и целых чисел

`bool` — это **целочисленный тип**.

Правила:

- `false` → 0
- `true` → 1

Любое ненулевое целое значение при приведении к `bool` становится `true`.

Примеры:

```cpp
bool a = 0;    // false
bool b = 1;    // true
bool c = 42;   // true
```

---

## 4️⃣ Арифметические операции с `bool`

⚠️ Хотя это выглядит странно, **арифметика с `bool` разрешена**, потому что `bool` — целочисленный тип.

### ➕ Сложение

```cpp
bool a = true;
bool b = true;

int x = a + b;  // 1 + 1 = 2
```

### ➖ Вычитание

```cpp
bool a = true;
bool b = false;

int x = a - b;  // 1 - 0 = 1
```

### ✖️ Умножение

```cpp
bool a = true;
bool b = false;

int x = a * b;  // 1 * 0 = 0
```

### ➗ Деление

```cpp
bool a = true;
bool b = true;

int x = a / b;  // 1 / 1 = 1
```

❌ Деление на `false`:

```cpp
bool a = true;
bool b = false;

int x = a / b;  // деление на 0 → неопределённое поведение
```

---

## 5️⃣ Результат арифметики

Важно понимать:

- результат арифметической операции с `bool` — **обычно `int`**, а не `bool`
- это связано с **целочисленными преобразованиями типов**

Пример:

```cpp
bool a = true;
bool b = true;

auto x = a + b;  // x имеет тип int
```

---

## 6️⃣ Логические vs арифметические операции

⚠️ Очень важное различие:

### Логические операции:

| Оператор | Описание       |
| -------- | -------------- |
| `&&`     | логическое И   |
|  \|\|    | логическое ИЛИ |
| `!`      | логическое НЕ  |

Пример:

```cpp
bool a = true;
bool b = false;

bool c = a && b; // false
```

### Арифметические / побитовые операции:

|Оператор|Описание|
|---|---|
|`+ - * /`|арифметика|
|`&|^`|

Пример:

```cpp
bool a = true;
bool b = false;

int x = a & b;  // 1 & 0 = 0
```

---

## 7️⃣ Неявные преобразования и подводные камни

```cpp
bool b = 2 + 3;   // true
bool c = -100;   // true
bool d = 0;      // false
```

Это часто приводит к логическим ошибкам, если думать, что `bool` может быть только `true` или `false` на уровне значений, но забывать о преобразованиях.

---

## 8️⃣ Когда арифметика с `bool` — плохая идея

Хотя разрешена:  
❌ ухудшает читаемость  
❌ легко приводит к ошибкам  
❌ не имеет логического смысла

Лучше писать:

```cpp
int count = (a ? 1 : 0) + (b ? 1 : 0);
```

или:

```cpp
int count = static_cast<int>(a) + static_cast<int>(b);
```

---

## 9️⃣ Итог

Надо понимать:

✔ `bool` хранит `true` или `false`  
✔ в памяти это 0 или 1  
✔ `bool` — целочисленный тип  
✔ арифметика с `bool` разрешена, но нежелательна  
✔ деление на `false` = деление на 0  
✔ для логики используем `&& || !`, а не арифметику
