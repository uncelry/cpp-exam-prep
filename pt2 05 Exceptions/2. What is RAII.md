# 1️⃣ Что такое RAII

## 📌 Определение

**RAII (Resource Acquisition Is Initialization)** — это идиома C++, при которой **владение ресурсом привязывается к времени жизни объекта**.

Расшифровка:

> **Resource Acquisition Is Initialization**  
> Получение ресурса происходит во время инициализации объекта.

Идея:

- ресурс захватывается в **конструкторе**
    
- ресурс освобождается в **деструкторе**
    

---

## Что считается ресурсом?

Ресурс — это всё, что нужно **явно освобождать**:

- память (`new`)
    
- файлы
    
- сокеты
    
- мьютексы
    
- дескрипторы ОС
    
- соединения с БД
    

---

# 2️⃣ Простейший пример RAII

Без RAII:

```cpp
void func() {
    FILE* f = fopen("data.txt", "r");

    if (!f)
        return;

    // работа с файлом

    fclose(f);
}
```

Проблема:  
если функция завершится раньше → файл не закроется.

---

## С RAII

```cpp
#include <fstream>

void func() {
    std::ifstream file("data.txt");

    // файл автоматически закроется в деструкторе
}
```

Когда объект `file` выходит из области видимости — вызывается деструктор.

---

# 3️⃣ Пример RAII вручную

```cpp
class File {
private:
    FILE* handle;

public:
    File(const char* name) {
        handle = fopen(name, "r");
    }

    ~File() {
        if (handle)
            fclose(handle);
    }
};
```

Использование:

```cpp
void func() {
    File f("data.txt");
}
```

При выходе из функции:

```
вызывается ~File()
```

---

# 4️⃣ Как RAII связан с исключениями

RAII делает код **безопасным при исключениях**.

Потому что при выбрасывании исключения происходит **stack unwinding**.

---

## Напоминание

При исключении:

- стек функций "раскручивается"
    
- все локальные объекты уничтожаются
    
- вызываются их деструкторы
    

---

## Пример

```cpp
#include <memory>
#include <stdexcept>

void func() {
    std::unique_ptr<int> ptr = std::make_unique<int>(5);

    throw std::runtime_error("error");
}
```

Что происходит:

1. выбрасывается исключение
    
2. начинается stack unwinding
    
3. вызывается деструктор `unique_ptr`
    
4. память освобождается
    

Утечки нет.

---

# 5️⃣ Что было бы без RAII

```cpp
void func() {
    int* ptr = new int(5);

    throw std::runtime_error("error");

    delete ptr; // никогда не выполнится
}
```

Происходит **утечка памяти**.

---

# 6️⃣ Почему RAII фундаментален для C++

Потому что:

- C++ активно использует **ручное управление ресурсами**
    
- RAII делает его **безопасным**
    

---

## RAII решает 3 ключевые проблемы

### 1️⃣ Утечки ресурсов

Ресурс всегда освобождается.

---

### 2️⃣ Исключения

RAII гарантирует корректное освобождение даже при:

- throw
    
- раннем return
    
- ошибках
    

---

### 3️⃣ Читаемость кода

Не нужно писать cleanup вручную.

---

# 8️⃣ Пример с mutex

Без RAII:

```cpp
mutex.lock();

doWork();

mutex.unlock();
```

Если возникнет исключение → mutex останется заблокирован.

---

## С RAII

```cpp
std::lock_guard<std::mutex> lock(mutex);

doWork();
```

Mutex автоматически разблокируется.

# 9️⃣ RAII и время жизни объекта

```
ресурс живёт ровно столько же,
сколько объект-владелец
```

# 1️⃣1️⃣ RAII и правило "нулевого управления"

Хороший C++ код:

- избегает `new`
    
- избегает `delete`
    
- использует RAII-классы
    

---

# 1️⃣2️⃣ Связь RAII и stack unwinding

Ключевая идея:

```
throw
 ↓
stack unwinding
 ↓
вызов деструкторов
 ↓
освобождение ресурсов
```

Именно поэтому RAII работает.

---

# 1️⃣3️⃣ RAII и безопасность исключений

RAII обеспечивает:

### Basic exception safety

Нет утечек ресурсов.

---

### Strong exception safety

Операция либо завершена, либо состояние не изменилось.

---

# 1️⃣4️⃣ Типичная реализация RAII

```cpp
class Resource {
public:
    Resource() {
        acquire();
    }

    ~Resource() {
        release();
    }
};
```

---

# 1️⃣5️⃣ Ограничения RAII

RAII работает только если:

- ресурс принадлежит объекту
    
- объект имеет корректный деструктор
    
- нет утечек владения
    

---

# 1️⃣6️⃣ Частые ошибки студентов

❌ Используют `new` вместо `unique_ptr`  
❌ Думают, что RAII только про память  
❌ Не понимают связь с исключениями  
❌ Путают RAII с garbage collection

# 1️⃣7️⃣ Минимальный экзаменационный пример

```cpp
#include <memory>

void func() {
    std::unique_ptr<int> p = std::make_unique<int>(10);
```

```
~unique_ptr() → delete
```

---

# 1️⃣8️⃣ Главное, что студент должен сказать на экзамене

RAII — это принцип:

- ресурс захватывается в **конструкторе**
    
- освобождается в **деструкторе**
    
- время жизни ресурса совпадает с временем жизни объекта
    

Он важен потому что:

- предотвращает утечки
    
- работает корректно при исключениях
    
- использует механизм stack unwinding
