## **Exceptions. How does the C++ exception model work? What happens when an exception is thrown? What is stack unwinding?**

## **Исключения. Как работает модель исключений в C++? Что происходит при выбрасывании исключения? Что такое раскрутка стека (stack unwinding)?**

---

# 1️⃣ Что такое исключения?

**Исключения (exceptions)** — это механизм обработки ошибок времени выполнения.

Вместо того чтобы возвращать коды ошибок, программа:

- генерирует (throw) объект ошибки
    
- передаёт управление ближайшему подходящему обработчику (catch)
    

---

## Базовый пример

```cpp
#include <iostream>
#include <stdexcept>

int divide(int a, int b) {
    if (b == 0)
        throw std::runtime_error("Division by zero");
    return a / b;
}

int main() {
    try {
        std::cout << divide(10, 0);
    }
    catch (const std::runtime_error& e) {
        std::cout << "Error: " << e.what();
    }
}
```

---

# 2️⃣ Как работает модель исключений в C++

Модель C++ построена на трёх ключевых словах:

- `try`
    
- `throw`
    
- `catch`
    

---

## Общий механизм

1. Выполняется код внутри `try`
    
2. Возникает `throw`
    
3. Выполняется поиск подходящего `catch`
    
4. Если найден — управление передаётся туда
    
5. Если не найден — вызывается `std::terminate()`
    

---

# 3️⃣ Что происходит при throw?

Рассмотрим:

```cpp
throw std::runtime_error("error");
```

Происходит следующее:

### 1️⃣ Создаётся объект исключения

(копируется во внутреннюю область памяти для исключений)

### 2️⃣ Прерывается текущий поток выполнения

Код после throw не выполняется

### 3️⃣ Начинается поиск обработчика

Сначала в текущей функции  
Потом выше по стеку вызовов

### 4️⃣ Запускается stack unwinding

---

# 4️⃣ Что такое stack unwinding?

## 📌 Определение

**Stack unwinding (раскрутка стека)** — это процесс уничтожения всех локальных объектов при выходе из функций во время распространения исключения.

Проще:

> Когда исключение "поднимается вверх", уничтожаются все объекты на пути.

---

## Пример

```cpp
#include <iostream>

struct A {
    ~A() {
        std::cout << "Destructor A\n";
    }
};

void func() {
    A a;
    throw std::runtime_error("error");
}

int main() {
    try {
        func();
    }
    catch (...) {
        std::cout << "Caught\n";
    }
}
```

### Вывод:

```
Destructor A
Caught
```

Объект `a` уничтожается автоматически.

---

# 5️⃣ Почему stack unwinding важен?

Это основа:

- RAII
    
- безопасного освобождения ресурсов
    
- автоматического закрытия файлов
    
- освобождения памяти
    

---

# 6️⃣ Исключения и RAII

Пример безопасного кода:

```cpp
#include <memory>

void func() {
    std::unique_ptr<int> ptr = std::make_unique<int>(5);
    throw std::runtime_error("error");
}
```

Память освободится автоматически.

---

# 7️⃣ Если исключение не поймано

Если подходящий `catch` не найден:

```cpp
std::terminate();
```

Программа аварийно завершается.

---

# 8️⃣ Важные детали экзамена

## 🔹 Исключения выбрасываются по значению

```cpp
throw MyException();
```

## 🔹 Ловятся по const reference

```cpp
catch (const MyException& e)
```

Почему?

- избегаем копирования
    
- сохраняем полиморфизм
    

---

# 9️⃣ Порядок catch блоков

```cpp
catch (const std::exception& e)
catch (const std::runtime_error& e)  // ❌ никогда не выполнится
```

Нужно ловить от более специфичного к более общему.

---

# 🔟 Повторный throw

```cpp
catch (...) {
    throw;  // повторно бросает текущее исключение
}
```

---

# 1️⃣1️⃣ Что нельзя делать

## ❌ Бросать исключение из деструктора

Если во время stack unwinding будет брошено второе исключение → `std::terminate()`.

---

## ❌ Игнорировать исключения

```cpp
catch (...) { }
```

Плохая практика.

---

# 1️⃣2️⃣ Полная схема работы

Представим стек:

```
main()
 └── f1()
      └── f2()
           └── f3() -> throw
```

Происходит:

1. f3() прекращается
    
2. Уничтожаются её локальные объекты
    
3. Переход к f2()
    
4. Уничтожаются объекты f2()
    
5. И так далее, пока не найден catch
    

---

# 1️⃣3️⃣ Виды исключений

### Стандартные (из `<stdexcept>`)

- `std::runtime_error`
    
- `std::logic_error`
    
- `std::invalid_argument`
    
- `std::out_of_range`
    

Все наследуются от:

```cpp
std::exception
```

# 1️⃣4️⃣ Пример пользовательского исключения

```cpp
#include <exception>

class MyException : public std::exception {
public:
    const char* what() const noexcept override {
        return "My custom error";
    }
};
```

# 1️⃣5️⃣ Гарантии безопасности исключений

В C++ есть уровни гарантий:

1️⃣ Basic guarantee — нет утечек  
2️⃣ Strong guarantee — операция либо успешна, либо ничего не изменилось  
3️⃣ No-throw guarantee — исключение не бросается

# 1️⃣6️⃣ Главное, что студент должен сказать на экзамене

- Исключения — механизм обработки ошибок runtime
    
- throw создаёт объект исключения
    
- Поиск catch идёт вверх по стеку
    
- При этом выполняется stack unwinding
    
- Деструкторы вызываются автоматически
    
- Ловить нужно по const reference
    
- Если исключение не поймано → terminate

# 1️⃣8️⃣ Минимальный экзаменационный пример

```cpp
#include <iostream>
#include <stdexcept>

void test() {
    throw std::runtime_error("Failure");
}

int main() {
    try {
        test();
    }
    catch (const std::exception& e) {
        std::cout << e.what();
    }
}
```
