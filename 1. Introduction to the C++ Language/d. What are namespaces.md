### 1. Namespaces

**Namespaces in C++ → Пространства имён в C++**

---

### 2. Что такое пространство имён

**Пространство имён (namespace)** — это механизм в C++, который позволяет группировать логически связанные сущности (переменные, функции, классы и т.д.) и **избегать конфликтов имён** между разными частями программы или библиотеками.

То есть это «контейнер имён», который помогает организовывать код.

---

### 3. Зачем они нужны

Основные цели пространств имён:

1️⃣ **Избежать конфликтов имён**  
Если в программе подключены разные библиотеки, в них могут быть функции или переменные с одинаковыми названиями. Без пространства имён возникнет ошибка.

2️⃣ **Упорядочить код**  
Можно логически разделять части программы:

- функции для математики
- функции для работы с файлами
- игровые объекты и т.д.

3️⃣ **Повысить читаемость**  
По имени пространства сразу понятно, к чему относится объект.

---

### 4. Как объявляется пространство имён

```
namespace MySpace {     
	int x = 10;      
	void print() {         
		std::cout << x << std::endl;     
	} 
}
```

Использование объектов из пространства:

```
MySpace::print(); 
std::cout << MySpace::x;
```

Оператор `::` называется **оператор разрешения области видимости**.

---

### 5. Пространство имён std

Почти вся стандартная библиотека C++ находится в пространстве имён `std`.

Пример:

```
std::cout << "Hello"; 
std::string name = "Alex";
```

Многие используют запись:

```
using namespace std;
```

чтобы писать:

```
cout << "Hello";
```

Но в современном C++ это считается **нежелательным**, особенно в больших проектах, потому что:

- может привести к конфликтам имён
- засоряет глобальную область имён

Лучше делать так:

```
using std::cout; 
using std::string;
```

или вообще всегда писать `std::`.

---

### 6. Вложенные пространства имён

Можно создавать пространства имён внутри других:

```
namespace A {     
	namespace B {         
		int value = 5;     
	} 
}
```

Использование:

```
std::cout << A::B::value;
```

В C++17 появилось упрощение:

```
namespace A::B {     
	int value = 5; 
}
```

---

### 7. Анонимное пространство имён

Если нужно ограничить использование объектов только в текущем файле, используют **анонимное пространство имён**:

```
namespace {     
	int counter = 0; 
}
```

Переменная `counter` теперь доступна только в этом файле (аналог `static` для глобальных переменных).

---

### 8. Итог для понимания студента

Пространства имён:

- помогают избежать конфликтов имён
- структурируют код
- активно используются стандартной библиотекой (`std`)
- позволяют удобно организовывать большие проекты
