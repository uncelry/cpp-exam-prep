## **What are class methods? Types of methods: regular methods, const methods, static methods, accessor methods (getters/setters). Difference between passing by const reference and declaring a method const.**

## **Что такое методы класса? Типы методов: обычные методы, const-методы, static-методы, методы доступа (геттеры/сеттеры). Разница между передачей по const-ссылке и объявлением метода const.**

---

# 1️⃣ Что такое методы класса?

**Методы класса (member functions)** — это функции, объявленные внутри класса и работающие с его данными.

Они:

- имеют доступ к private и protected членам
    
- неявно получают указатель `this` (кроме static)
    
- определяют поведение объекта
    

---

## Пример

```cpp
class Person {
private:
    std::string name;

public:
    void setName(const std::string& n) {
        name = n;
    }

    std::string getName() const {
        return name;
    }
};
```

---

# 2️⃣ Regular methods — обычные методы

## 📌 Определение

Обычные методы:

- работают с объектом
    
- имеют доступ к `this`
    
- могут изменять состояние объекта
    

---

## Пример

```cpp
class Counter {
private:
    int value = 0;

public:
    void increment() {
        ++value;   // изменяет состояние
    }
};
```

---

## Что важно понимать

Компилятор фактически превращает:

```cpp
obj.increment();
```

в

```cpp
Counter::increment(&obj);
```

То есть передаётся `this`.

---

# 3️⃣ Const methods — константные методы

## 📌 Определение

Метод с `const` после сигнатуры:

```cpp
int getValue() const;
```

означает:

> Этот метод НЕ изменяет состояние объекта.

---

## Пример

```cpp
class Counter {
private:
    int value = 0;

public:
    int getValue() const {
        return value;  // разрешено
    }
};
```

---

## Что запрещено внутри const метода?

```cpp
int getValue() const {
    value++;   // ❌ ошибка
}
```

Потому что внутри такого метода:

```cpp
this -> const Counter*
```

---

## Почему это важно?

- Можно вызывать метод у const-объектов
    
- Защищает от случайного изменения состояния
    

---

## Пример

```cpp
const Counter c;
c.getValue();   // ✅ работает
```

Если метод не const — вызов невозможен.

---

# 4️⃣ Static methods — статические методы

## 📌 Определение

Static метод:

- принадлежит классу, а не объекту
    
- не имеет `this`
    
- может работать только со static членами
    

---

## Пример

```cpp
class Math {
public:
    static int add(int a, int b) {
        return a + b;
    }
};
```

Вызов:

```cpp
Math::add(3, 4);
```

---

## Ограничения

❌ Нельзя обращаться к нестатическим полям  
❌ Нет `this`  
❌ Не может быть virtual

---

# 5️⃣ Accessor methods — геттеры и сеттеры

## 📌 Определение

Методы для контролируемого доступа к private полям.

---

## Getter

```cpp
int getAge() const {
    return age;
}
```

Обычно:

- const
    
- возвращает значение или const ссылку
    

---

## Setter

```cpp
void setAge(int a) {
    if (a > 0)
        age = a;
}
```

Позволяет:

- проверять данные
    
- поддерживать инварианты
    

---

## Почему не делать поле public?

Потому что:

- нет контроля
    
- нельзя добавить логику
    
- ломается инкапсуляция
    

---

# 6️⃣ Важная часть — разница между:

### 🔹 Passing by const reference

и

### 🔹 Declaring a method const

Это разные вещи.

---

# 🔹 Passing by const reference

```cpp
void setName(const std::string& n);
```

Означает:

> Мы не будем изменять параметр.

Это про **аргумент функции**.

---

## Зачем?

- избегаем копирования
    
- защищаем аргумент
    

---

# 🔹 Declaring a method const

```cpp
std::string getName() const;
```

Означает:

> Мы не будем изменять объект.

Это про **состояние самого объекта**.

---

# 📌 Ключевая разница

|Passing const reference|Method const|
|---|---|
|Защищает аргумент|Защищает объект|
|`const T& param`|`method() const`|
|Относится к параметру|Относится к this|

---

# 7️⃣ Полный пример

```cpp
class Person {
private:
    std::string name;

public:
    void setName(const std::string& n) {  // const reference
        name = n;
    }

    const std::string& getName() const {  // method const
        return name;
    }
};
```

Здесь:

- `const std::string& n` — защита аргумента
    
- `getName() const` — защита объекта
    

---

# 8️⃣ Частые ошибки студентов

❌ Забывают ставить const у геттеров  
❌ Думают, что const метод делает объект полностью неизменяемым  
❌ Путают const у параметра и const у метода  
❌ Пытаются менять поля внутри const метода

---

# 9️⃣ Экзаменационный tricky-вопрос

Почему этот код не компилируется?

```cpp
void print(const Person& p) {
    p.getName();
}
```

Если `getName()` не объявлен как `const`.

Ответ:

Потому что `p` — const объект,  
а не-const метод нельзя вызывать у const объекта.

# 🔟 Минимальный пример, который студент должен уметь написать

```cpp
#include <iostream>
#include <string>

class Student {
private:
    std::string name;
    int age;

public:
    Student(const std::string& n, int a)
        : name(n), age(a) {}

    const std::string& getName() const {
        return name;
    }

    int getAge() const {
        return age;
    }

    void setAge(int a) {
        if (a > 0)
            age = a;
    }

    static void printSchool() {
        std::cout << "C++ Academy\n";
    }
};
```

---

# 🔴 Главное, что студент должен сказать на экзамене

- Метод класса — функция, работающая с данными класса
    
- Regular метод может менять объект
    
- Const метод гарантирует неизменяемость объекта
    
- Static метод не имеет this
    
- Getter — обычно const
    
- Setter — проверяет данные
    
- const у параметра ≠ const у метода
