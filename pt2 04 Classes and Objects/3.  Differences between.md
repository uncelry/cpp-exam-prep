## **Differences between public, private, and protected. Examples.**

## **Различия между public, private и protected. Примеры.**

Это спецификаторы доступа в C++, которые управляют тем, **кто и откуда может обращаться к членам класса**.

Они являются частью принципа **инкапсуляции (encapsulation)** в ООП.

---

# 1️⃣ private

## 📌 Определение

`private` — доступ разрешён **только внутри самого класса** (и его friends).

Ни снаружи, ни в наследниках напрямую обратиться нельзя.

---

## 📌 Пример

```cpp
class Person {
private:
    int age;

public:
    Person(int a) : age(a) {}

    int getAge() const {
        return age;
    }
};
```

```cpp
int main() {
    Person p(25);

    // p.age = 30;  ❌ ошибка компиляции
}
```

---

## 📌 Что важно объяснить студенту

- Это основной инструмент инкапсуляции
    
- Поля почти всегда должны быть private
    
- Контроль доступа осуществляется через public методы
    

---

# 2️⃣ public

## 📌 Определение

`public` — доступ разрешён **откуда угодно**:

- из main
    
- из других классов
    
- из наследников
    

---

## 📌 Пример

```cpp
class Example {
public:
    int x;
};

int main() {
    Example e;
    e.x = 10;  // ✅ доступ разрешён
}
```

---

## 📌 Когда используется

- Интерфейс класса
    
- Методы, которые должен видеть пользователь класса
    

---

# 3️⃣ protected

## 📌 Определение

`protected` — доступ разрешён:

- внутри самого класса
    
- в производных классах
    
- но НЕ из внешнего кода
    

---

## 📌 Пример

```cpp
class Base {
protected:
    int value = 42;
};

class Derived : public Base {
public:
    void print() {
        std::cout << value << "\n";  // ✅ доступ есть
    }
};

int main() {
    Derived d;

    // d.value;  ❌ ошибка
}
```

---

# 📊 Сравнение

|Спецификатор|Внутри класса|В наследнике|Снаружи|
|---|---|---|---|
|private|✅|❌|❌|
|protected|✅|✅|❌|
|public|✅|✅|✅|

---

# 4️⃣ Влияние типа наследования

Очень важный экзаменационный момент.

```cpp
class Derived : public Base
class Derived : protected Base
class Derived : private Base
```

Тип наследования меняет уровень доступа унаследованных членов.

---

## 🔹 public inheritance

- public остаётся public
    
- protected остаётся protected
    

---

## 🔹 protected inheritance

- public → protected
    
- protected → protected
    

---

## 🔹 private inheritance

- public → private
    
- protected → private
    

---

# 5️⃣ Пример со всеми тремя

```cpp
class Base {
public:
    int a = 1;
protected:
    int b = 2;
private:
    int c = 3;
};

class Derived : public Base {
public:
    void test() {
        std::cout << a;  // ✅
        std::cout << b;  // ✅
        // std::cout << c; ❌
    }
};
```

---

# 6️⃣ Почему protected используется реже?

Потому что:

- нарушает инкапсуляцию
    
- жёстко связывает базовый и производный класс
    
- усложняет поддержку
    

Современный стиль чаще предпочитает:

- private + protected методы доступа
    
- композицию вместо наследования
    

---

# 7️⃣ Частые ошибки студентов

❌ Думают, что protected доступен снаружи  
❌ Думают, что private доступен наследнику  
❌ Путают protected и private inheritance  
❌ Делают поля public

---

# 8️⃣ Правило хорошего стиля

### ✔ Поля → private

### ✔ Интерфейс → public

### ✔ protected — только если действительно нужно наследникам

---

# 9️⃣ Минимальный экзаменационный пример

```cpp
#include <iostream>

class Account {
private:
    double balance;

protected:
    void log() const {
        std::cout << "Balance accessed\n";
    }

public:
    Account(double b) : balance(b) {}

    double getBalance() const {
        log();
        return balance;
    }
};

class SavingsAccount : public Account {
public:
    SavingsAccount(double b) : Account(b) {}

    void show() {
        log();  // protected доступен
    }
};

int main() {
    SavingsAccount acc(1000);

    std::cout << acc.getBalance() << "\n";

    // acc.balance; ❌
    // acc.log();   ❌
}
```

---

# 🔟 Главное, что студент должен сказать на экзамене

- `private` — скрывает реализацию
    
- `protected` — доступно наследникам
    
- `public` — интерфейс класса
    
- Тип наследования влияет на уровень доступа
    
- Это механизм реализации инкапсуляции
