Разберём тему системно: **классы, объекты, поля, методы и расположение объекта в памяти (memory layout)**.

---

# 1️⃣ Что такое класс?

**Класс (class)** — это пользовательский тип данных, который:

- объединяет данные (поля)
    
- и функции (методы)
    
- задаёт структуру и поведение объектов
    

Это **шаблон (чертёж)** для создания объектов.

---

## Пример

```cpp
class Person {
    std::string name;   // поле
    int age;            // поле

public:
    void print() {      // метод
        std::cout << name << " " << age;
    }
};
```

---

# 2️⃣ Что такое объект?

**Объект** — это конкретный экземпляр класса, размещённый в памяти.

```cpp
Person p;
```

- `Person` — тип
    
- `p` — объект
    
- для `p` выделяется память
    

---

# 3️⃣ Поля (Fields, Data Members)

Поля — это переменные внутри класса.

```cpp
class A {
    int x;
    double y;
};
```

Каждый объект `A` содержит:

- своё `x`
    
- своё `y`
    

---

# 4️⃣ Методы (Methods)

Методы — это функции, принадлежащие классу.

```cpp
class A {
public:
    void foo() {}
};
```

Важно:

> Методы НЕ хранятся внутри каждого объекта.

Они существуют в кодовой области программы (text segment).

---

# 5️⃣ Объект в памяти

Рассмотрим:

```cpp
class A {
    int x;
    char c;
};
```

Если:

- `int` = 4 байта
    
- `char` = 1 байт
    

Интуитивно ожидаем 5 байт.

Но реально будет больше из-за выравнивания (alignment).

---

# 6️⃣ Physical order of fields

Очень важное правило:

> Поля располагаются в памяти в порядке их объявления в классе.

Например:

```cpp
class A {
    int x;
    char c;
    double d;
};
```

Порядок в памяти:

```
x → c → padding → d
```

Компилятор НЕ меняет порядок полей.

---

# 7️⃣ Выравнивание (Alignment)

Каждый тип имеет требование выравнивания.

Например:

- int → 4 байта
    
- double → 8 байт
    

Чтобы `double` начинался по адресу, кратному 8, компилятор добавляет padding.

---

## Пример

```cpp
class A {
    char c;     // 1 байт
    double d;   // 8 байт
};
```

Память:

```
c
padding (7 байт)
d
```

Размер будет 16 байт, а не 9.

---

# 8️⃣ Размер объекта (sizeof)

Размер объекта:

- включает все поля
    
- включает padding
    
- не включает методы
    
- может включать скрытые поля (например, vptr)
    

Пример:

```cpp
sizeof(A)
```

---

# 9️⃣ Влияние порядка полей на размер

Плохой порядок:

```cpp
class A {
    char c;
    double d;
    int x;
};
```

Лучший порядок:

```cpp
class A {
    double d;
    int x;
    char c;
};
```

Правильная группировка уменьшает padding.

---

# 🔟 Виртуальные функции и vptr

Если есть virtual-функция:

```cpp
class A {
public:
    virtual void foo();
};
```

В объект добавляется скрытый указатель:

```
vptr
```

Он указывает на таблицу виртуальных функций (vtable).

Это увеличивает размер объекта (обычно на размер указателя).

---

# 1️⃣1️⃣ Пример полного layout

```cpp
class A {
    int x;
    virtual void foo();
};
```

В памяти:

```
vptr (8 байт)
x (4 байта)
padding (4 байта)
```

---

# 1️⃣2️⃣ Важные свойства layout

✔ Поля идут в порядке объявления  
✔ Методы не занимают место в объекте  
✔ Размер зависит от alignment  
✔ Virtual добавляет vptr  
✔ Пустой класс имеет размер ≥ 1

---

# 1️⃣3️⃣ Почему пустой класс не равен 0 байт?

```cpp
class Empty {};
```

Размер будет 1 байт.

Причина:

- два объекта должны иметь разные адреса
    

---

# 1️⃣4️⃣ Краткое резюме

### Класс:

- пользовательский тип
    
- объединяет данные и поведение
    

### Объект:

- экземпляр класса
    
- занимает память
    

### Layout:

- поля располагаются в порядке объявления
    
- компилятор добавляет padding
    
- методы не занимают место
    
- virtual добавляет vptr
    
- размер зависит от alignment
