## **Name and explain the principles of OOP: encapsulation, inheritance, polymorphism, abstraction. Why is each needed? Examples.**

## **Назовите и объясните принципы ООП: инкапсуляция, наследование, полиморфизм, абстракция. Зачем каждый из них нужен? Примеры.**

---

Объектно-ориентированное программирование (OOP) строится на четырёх базовых принципах:

1. **Encapsulation — Инкапсуляция**
    
2. **Inheritance — Наследование**
    
3. **Polymorphism — Полиморфизм**
    
4. **Abstraction — Абстракция**
    

Это фундаментальные идеи, которые лежат в основе проектирования программ.

---

# 1️⃣ Encapsulation — Инкапсуляция

## 📌 Определение

**Инкапсуляция** — это объединение данных и методов, работающих с этими данными, в одном объекте, с одновременным сокрытием внутренней реализации.

Проще:

> Объект сам управляет своим состоянием, а доступ к данным ограничен.

---

## 📌 Как реализуется в C++?

Через:

- `private`
    
- `protected`
    
- `public`
    

---

## 📌 Пример

```cpp
class BankAccount {
private:
    double balance;  // скрытое поле

public:
    BankAccount(double initial) : balance(initial) {}

    void deposit(double amount) {
        if (amount > 0)
            balance += amount;
    }

    double getBalance() const {
        return balance;
    }
};
```

Пользователь не может напрямую изменить `balance`.

---

## 📌 Зачем нужна инкапсуляция?

### 1️⃣ Защита данных

Нельзя случайно испортить внутреннее состояние.

### 2️⃣ Контроль инвариантов

Можно проверять корректность данных.

### 3️⃣ Снижение связанности

Внешний код не зависит от внутренней реализации.

---

## 📌 Типичный вопрос на экзамене

> Почему делать поля public — плохая практика?

Потому что:

- теряется контроль
    
- нарушаются инварианты
    
- сложно менять реализацию
    

---

# 2️⃣ Inheritance — Наследование

## 📌 Определение

**Наследование** — это механизм создания нового класса на основе существующего.

> Производный класс получает поведение базового класса.

---

## 📌 Пример

```cpp
class Animal {
public:
    void eat() {
        std::cout << "Eating\n";
    }
};

class Dog : public Animal {
public:
    void bark() {
        std::cout << "Woof\n";
    }
};
```

`Dog` наследует `eat()`.

---

## 📌 Зачем нужно наследование?

### 1️⃣ Повторное использование кода

Не нужно переписывать одинаковую логику.

### 2️⃣ Расширение функциональности

Можно добавить новое поведение.

### 3️⃣ Иерархия типов

Позволяет строить "is-a" отношения:

```
Dog is an Animal
```

---

## 📌 Важно

Наследование — это не просто "копирование кода".  
Это моделирование иерархии.

---

# 3️⃣ Polymorphism — Полиморфизм

## 📌 Определение

**Полиморфизм** — это способность объектов разных типов обрабатываться через общий интерфейс.

Проще:

> Один интерфейс — разные реализации.

---

## 📌 Виды полиморфизма в C++

### 1️⃣ Compile-time (перегрузка)

```cpp
int add(int a, int b);
double add(double a, double b);
```

### 2️⃣ Runtime (через virtual)

Это главный принцип ООП.

---

## 📌 Пример runtime polymorphism

```cpp
class Animal {
public:
    virtual void speak() const {
        std::cout << "Animal sound\n";
    }
};

class Dog : public Animal {
public:
    void speak() const override {
        std::cout << "Woof\n";
    }
};

void makeSound(const Animal& a) {
    a.speak();
}
```

Теперь:

```cpp
Dog d;
makeSound(d);  // вызовется Dog::speak()
```

---

## 📌 Зачем нужен полиморфизм?

### 1️⃣ Работа через базовый интерфейс

Можно писать универсальный код.

### 2️⃣ Расширяемость

Добавление новых типов не требует переписывания старого кода.

### 3️⃣ Принцип Open/Closed

Код открыт для расширения, закрыт для изменения.

---

# 4️⃣ Abstraction — Абстракция

## 📌 Определение

**Абстракция** — это выделение только существенных характеристик объекта и сокрытие деталей реализации.

> Мы описываем ЧТО делает объект, а не КАК.

---

## 📌 Реализация в C++

Через:

- интерфейсы
    
- абстрактные классы
    
- чисто виртуальные функции
    

---

## 📌 Пример

```cpp
class Shape {
public:
    virtual double area() const = 0;  // чисто виртуальная функция
};
```

Это абстрактный класс.

Производный класс обязан реализовать:

```cpp
class Circle : public Shape {
private:
    double radius;

public:
    Circle(double r) : radius(r) {}

    double area() const override {
        return 3.14 * radius * radius;
    }
};
```

---

## 📌 Зачем нужна абстракция?

### 1️⃣ Упрощение модели

Работаем на уровне концепций.

### 2️⃣ Снижение сложности

Пользователь класса не видит внутреннюю реализацию.

### 3️⃣ Гибкость

Можно менять реализацию без изменения интерфейса.

---

# 📊 Как связаны эти принципы?

|Принцип|Что решает|
|---|---|
|Инкапсуляция|Защита данных|
|Наследование|Повторное использование|
|Полиморфизм|Гибкость поведения|
|Абстракция|Упрощение модели|

---

# 📌 Очень важное понимание

### Инкапсуляция — про данные

### Наследование — про структуру

### Полиморфизм — про поведение

### Абстракция — про уровень модели

---

# 📌 Частые ошибки студентов

❌ Путают абстракцию и инкапсуляцию  
❌ Думают, что ООП = наследование  
❌ Не понимают разницу между перегрузкой и полиморфизмом времени выполнения  
❌ Не используют virtual

# 📌 Минимальный пример, объединяющий всё

```cpp
#include <iostream>
#include <vector>
#include <memory>

class Shape {
public:
    virtual double area() const = 0;
    virtual ~Shape() = default;
};

class Rectangle : public Shape {
private:
    double width, height;

public:
    Rectangle(double w, double h) : width(w), height(h) {}

    double area() const override {
        return width * height;
    }
};

class Circle : public Shape {
private:
    double radius;

public:
    Circle(double r) : radius(r) {}

    double area() const override {
        return 3.14 * radius * radius;
    }
};

int main() {
    std::vector<std::unique_ptr<Shape>> shapes;

    shapes.push_back(std::make_unique<Rectangle>(3, 4));
    shapes.push_back(std::make_unique<Circle>(5));

    for (const auto& s : shapes)
        std::cout << s->area() << "\n";
}
```

Здесь есть:

- абстракция (Shape)
    
- инкапсуляция (private поля)
    
- наследование (Rectangle, Circle)
    
- полиморфизм (virtual area)
